# Competition Checklist

## Submission-level goals

The competition version should make these points obvious within the first minute:

1. This is not just a CLI tool.
2. The real core is an organizational intelligence layer.
3. The project already has a running analysis engine.
4. The project now has a Skill packaging layer.
5. The project has a believable Feishu landing path.

## What the repo should communicate

### Homepage / README
- clearly explain the project as organizational intelligence
- explain why CLI was phase 1 instead of the final form
- explain the Skill packaging layer
- show how outputs flow into Feishu Docs / Base / Chat
- show demo steps in under 1 minute

### Skill layer
- clear use cases
- clear capability list
- clear input / output contract
- clear next-step Feishu integration plan

### Demo assets
- one sample data file
- one sample weekly brief output
- one sample risk scan output
- one docs-style output artifact

## Recommended demo story

1. Structured world model exists
2. Intelligence layer scans and interprets execution state
3. Skill layer packages this as callable capability
4. Output lands back into Feishu work surfaces
5. This points to a future organizational operating system inside Feishu
