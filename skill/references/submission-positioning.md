# Submission Positioning

本项目提交时应被定义为：

> 一个以 Skill 形态交付的组织智能系统原型，而不是单纯的 CLI 工具。

核心论点：

1. CLI 只是第一阶段控制台，不是最终产品形态。
2. 真正的核心是世界模型 + intelligence layer + Feishu output layer。
3. Skill 是赛事要求下最合理的交付包装层，因为它能把“能力”而非“命令行程序”暴露给评委与使用者。
4. 项目应突出“从 hierarchy 到 intelligence”的组织协作升级，而不是突出 Node.js / TypeScript 实现本身。
